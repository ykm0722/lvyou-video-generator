# 旅游视频生成系统 - 部署指南

> 本指南适合零基础小白，按步骤操作即可完成部署

## 📋 项目介绍

这是一个自动生成旅游推广视频的系统：
- **前端**：网页界面，用户上传PDF和图片
- **后端**：处理数据，生成视频
- **部署方案**：完全免费的云服务

## 🎯 部署架构

```
前端（EdgeOne Pages） ←→ 后端（Render）
     ↓                        ↓
  用户访问                 处理业务逻辑
```

## ⚠️ 部署前准备

### 1. 注册账号

需要注册以下账号（都是免费的）：

1. **GitHub账号**（代码托管）
   - 访问：https://github.com
   - 点击右上角 Sign up 注册

2. **Render账号**（后端部署）
   - 访问：https://render.com
   - 点击 Get Started 注册
   - 建议用GitHub账号直接登录

3. **腾讯云账号**（前端部署）
   - 访问：https://cloud.tencent.com
   - 点击右上角注册
   - 需要实名认证（上传身份证）

### 2. 准备API密钥

需要获取以下API密钥：

1. **Anthropic API Key**（必需，用于AI生成内容）
   - 访问：https://console.anthropic.com
   - 注册并创建API密钥
   - 或使用第三方代理服务

2. **OpenAI API Key**（可选，用于语音合成）
   - 访问：https://platform.openai.com
   - 创建API密钥

3. **Pexels API Key**（可选，用于图片搜索）
   - 访问：https://www.pexels.com/api
   - 免费注册并获取密钥

## 🚀 第一步：推送代码到GitHub

### 1.1 创建GitHub仓库

1. 登录GitHub
2. 点击右上角 `+` → `New repository`
3. 填写信息：
   - Repository name: `lvyou-video-generator`
   - Description: `旅游视频生成系统`
   - 选择 `Public`（公开）
4. 点击 `Create repository`

### 1.2 推送代码

在项目目录打开终端，执行：

```bash
# 初始化git（如果还没有）
git init

# 添加远程仓库（替换YOUR_USERNAME为你的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/lvyou-video-generator.git

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 推送到GitHub
git push -u origin main
```

✅ **检查点**：访问你的GitHub仓库页面，应该能看到所有代码文件

## 🔧 第二步：部署后端到Render

### 2.1 创建Web Service

1. 登录 https://render.com
2. 点击右上角 `New +` → 选择 `Web Service`
3. 点击 `Connect GitHub`，授权Render访问你的GitHub
4. 在仓库列表中找到 `lvyou-video-generator`，点击 `Connect`

### 2.2 配置服务

填写以下配置：

| 配置项 | 填写内容 |
|--------|----------|
| **Name** | `lvyou-backend`（或任意名称） |
| **Region** | 选择 `Singapore`（离中国最近，速度快） |
| **Branch** | `main` |
| **Root Directory** | `backend` ⚠️ 重要！ |
| **Runtime** | `Python 3` |
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `uvicorn app:app --host 0.0.0.0 --port $PORT` |

### 2.3 选择免费套餐

1. 向下滚动到 `Instance Type`
2. 选择 `Free`（免费套餐）
3. 注意：免费套餐15分钟无活动会休眠，首次访问需等待30秒唤醒

### 2.4 配置环境变量

1. 点击 `Advanced` 展开高级选项
2. 点击 `Add Environment Variable` 添加以下变量：

| Key | Value | 说明 |
|-----|-------|------|
| `ANTHROPIC_API_KEY` | 你的Claude API密钥 | 必需 |
| `ANTHROPIC_BASE_URL` | API代理地址（如有） | 可选 |
| `OPENAI_API_KEY` | 你的OpenAI密钥 | 可选 |
| `OPENAI_BASE_URL` | API代理地址（如有） | 可选 |
| `PEXELS_API_KEY` | 你的Pexels密钥 | 可选 |

### 2.5 开始部署

1. 点击页面底部 `Create Web Service`
2. 等待部署完成（约3-5分钟）
3. 部署成功后，页面顶部会显示你的后端地址，例如：
   ```
   https://lvyou-backend.onrender.com
   ```
4. **复制这个地址**，后面配置前端时需要用到

✅ **检查点**：
- 访问后端地址，应该看到 `{"message":"Travel Video Generator API"}`
- 访问 `后端地址/docs`，应该看到API文档页面

## 🎨 第三步：部署前端到EdgeOne Pages

### 3.1 登录腾讯云

1. 访问 https://console.cloud.tencent.com
2. 登录你的腾讯云账号
3. 在搜索框输入 `EdgeOne Pages`，点击进入

### 3.2 创建站点

1. 点击 `新建站点`
2. 选择 `从Git仓库导入`
3. 点击 `授权GitHub账号`，允许腾讯云访问你的GitHub

### 3.3 选择仓库

1. 在仓库列表中选择 `lvyou-video-generator`
2. 点击 `导入`

### 3.4 配置构建

填写以下配置：

| 配置项 | 填写内容 |
|--------|----------|
| **站点名称** | `lvyou-video-generator`（或任意名称） |
| **框架预设** | 选择 `Vue` |
| **根目录** | `frontend` ⚠️ 重要！ |
| **构建命令** | `npm install && npm run build` |
| **输出目录** | `dist` |

### 3.5 配置环境变量

1. 点击 `环境变量` 标签
2. 添加环境变量：

| Key | Value |
|-----|-------|
| `VITE_API_BASE_URL` | 你的Render后端地址（第二步复制的） |

例如：`https://lvyou-backend.onrender.com`

### 3.6 开始部署

1. 点击 `部署`
2. 等待构建完成（约2-3分钟）
3. 部署成功后，会显示你的前端访问地址，例如：
   ```
   https://lvyou-video-generator.zh-cn.edgeone.cool
   ```

✅ **检查点**：访问前端地址，应该看到漂亮的网页界面

## ❓ 常见问题

### Q1: 后端部署失败，显示 "Build failed"

**原因**：可能缺少依赖包或配置错误

**解决**：
1. 检查 `backend/requirements.txt` 是否包含所有依赖
2. 检查Render配置中的 Root Directory 是否设置为 `backend`
3. 查看Render的部署日志，找到具体错误信息

### Q2: 前端访问显示 401 或 403 错误

**原因**：EdgeOne Pages的访问控制设置

**解决**：
1. 进入EdgeOne Pages控制台
2. 找到你的站点
3. 点击 "设置" → "访问控制"
4. 确保设置为 "公开访问"

### Q3: 前端可以访问，但点击"开始智能处理"报错

**原因**：前端无法连接后端

**解决**：
1. 检查前端环境变量 `VITE_API_BASE_URL` 是否正确
2. 检查后端是否正常运行（访问后端地址）
3. 检查浏览器控制台的错误信息（按F12打开）
4. 可能是后端休眠了，等待30秒重试

### Q4: 后端显示 "No module named 'xxx'"

**原因**：缺少Python依赖包

**解决**：
1. 在 `backend/requirements.txt` 中添加缺失的包
2. 提交代码到GitHub
3. Render会自动重新部署

### Q5: 视频生成失败

**原因**：可能是API密钥无效或余额不足

**解决**：
1. 检查Render环境变量中的API密钥是否正确
2. 检查API账户余额是否充足
3. 查看Render的日志，找到具体错误信息

### Q6: 免费套餐的限制

**Render免费套餐限制**：
- 15分钟无活动会休眠
- 每月750小时运行时间
- 首次访问需要30秒唤醒

**EdgeOne Pages免费套餐**：
- 每月100GB流量
- 无限次构建
